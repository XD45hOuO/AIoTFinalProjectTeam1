智慧電表 異常用電偵測系統

基於非監督式機器學習的 AIoT 電力異常分析

本專案為 AIoT 課程專案，旨在解決智慧電表普及後，海量數據難以透過人工即時排查異常用電（如竊電、設備故障）的痛點。
我們利用非監督式機器學習演算法，打造一套不需事先標記資料的異常偵測與視覺化系統。

--------------------------------------------------------------------------------
🛠 開發方法：Spec-Driven Development

本專案採用 Agent 規格驅動開發 (Spec-Driven Development) 模式，整體分為 Phase 1 與 Phase 2 兩大階段：
規格產出 (Phase 1)：將提案的 PPT 檔案轉換為 PDF，接著透過 NBLM Agent (結合 Antigravity 與 Openspec) 自動產出包含架構與資訊圖表的 PDR 報告 (產品設計規格書，約 12 頁)。
實作與驗證循環 (Phase 1 & 2)：依據產出的 PDR 生成 Task.md，進一步轉換為實際 Code，並在開發過程中透過「Plan (計畫) ➔ Apply (應用) ➔ Verify (驗證)」的迭代循環來確保系統品質。

--------------------------------------------------------------------------------
💡 背景與動機

痛點：非技術性損失 (NTL) 佔電力損失九成以上，全球每年損失高達約 $101.2B。

現況：台灣已安裝約 423 萬具智慧電表，但缺乏自動化分析異常的功能。傳統依賴人工抄表與事後稽查，往往具有數日至數月的滯後性，導致損失難以追回。

目標：利用 AI 技術即時偵測海量電力數據中的異常，大幅提高即時性並降低人工稽查成本。

--------------------------------------------------------------------------------
🏗 系統架構

系統整體設計分為三個層次，實現端對端的異常偵測流程：

1. 資料層 (Data Layer)
模擬真實 IoT 智慧電表資料流，採用公開免費的 UCI Electricity Load Diagrams (2011–2014) 資料集。

2. 運算層 (Processing / AI Model)
資料前處理：使用前向填補處理缺失值 (缺失率 < 0.3%)，並透過 MinMaxScaler 進行數值正規化。
特徵工程：萃取時間規律 (如 hour, day_of_week)、Lag 與 Rolling Statistics 等特徵。
模型訓練 (Isolation Forest)：採用孤立森林演算法，透過計算資料點被隔離的路徑長度來判斷異常 (路徑越短代表越異常)。此方法不需標記資料且硬體需求低。
合成異常注入驗證：為了計算精確率 (Precision)、召回率 (Recall) 與 F1-Score，我們設定固定的 random seed (42)，並隨機注入 10 個合成異常 (包含 5 倍突波與 0.1 倍驟降) 作為量化評估標準。

3. 應用層 (App Layer / Dashboard)
使用 Streamlit 打造視覺化介面 (運行於 Localhost:8501)。
在時序圖上即時標示異常點 (紅色標記)，讓稽查人員能直觀定位問題，並可匯出異常事件 Log 供後續追蹤。

💻 系統規格與技術棧

程式語言：Python 3.10+

主要套件：pandas, scikit-learn, streamlit, matplotlib

AI 模型：Isolation Forest (contamination=0.05, random_state=42)

--------------------------------------------------------------------------------
📈 實驗結果

在沒有使用任何標記資料的情況下，本專案使用的 Isolation Forest 模型，在 F1-Score 的綜合表現上，成功優於傳統的 Z-Score 門檻與移動平均 (3σ) 規則式方法。
不僅減少了假警報，也能更有效適應複雜的用電行為。

🌟 未來展望 (Future Work)

特徵工程優化：計畫引入更多外部變數（如天氣資料、節慶假日等）來進一步提升預測準確度。

AI 模型升級：嘗試引入 LSTM Autoencoder 以強化時序建模能力，並與半監督式方法進行比較研究。

邊緣運算部署 (Edge Computing)：將模型部署至真實邊緣設備（如 Raspberry Pi）上進行實際測試，降低傳輸延遲與提升隱私保護。

自動化通知：串接 n8n 與 LINE Bot，在偵測到異常時實現自動推播告警。
